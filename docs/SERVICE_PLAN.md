# 타래 서비스 기획안

---

## 1. 서비스 개요

**서비스명:** 타래 (Tarae)

**서비스 설명:**

타래는 뜨개 관련 장소, 정보, 일정, 사람을 한 곳에서 탐색할 수 있는 플랫폼이다.

흩어져 있는 뜨개 정보를 지도, 캘린더, 데이터 구조로 통합하여 사용자에게 직관적인 탐색 경험을 제공한다.

**서비스 비전:**

뜨개 산업에 산재된 정보를 구조화하여

"어디서, 무엇을, 언제, 누구와 뜰 것인가"를 하나의 서비스에서 해결한다.

---

## 2. 문제 정의

현재 뜨개 관련 정보는 다음과 같은 문제를 가진다.

- 정보가 여러 플랫폼에 분산되어 있음 (커뮤니티, SNS, 쇼핑몰, 지도 등)
- 위치 기반 탐색이 어려움
- 최신 정보 (이전, 폐업, 일정 등) 확인이 어려움
- 실, 도안, 후기 정보가 구조화되어 있지 않음
- 사용자 간 연결 (모임, 교류)이 어려움

결과적으로 사용자는 여러 플랫폼을 반복적으로 오가며 정보를 수집해야 한다.

---

## 3. 해결 전략

타래는 다음 4가지 축으로 문제를 해결한다.

1. **장소 (Where)** — 지도 기반으로 뜨개 관련 장소 탐색
2. **일정 (When)** — 캘린더 기반으로 이벤트, 세일, 모집 정보 제공
3. **데이터 (What)** — 실, 도안, 후기, 게이지 등 구조화된 정보 제공
4. **사람 (Who)** — 함뜨 및 커뮤니티를 통해 사용자 연결

---

## 4. 서비스 구조

### 탭1: 지도 (Places)

- 뜨개샵, 공방, 뜨개카페 등 장소 탐색
- 지도 기반 UI
- 마커 및 필터 제공
- 핵심 MVP 기능

### 탭2: 캘린더 (Events)

**기본 구조:**

- 캘린더 기반 UI
- 필터로 카테고리별 일정 확인

**포함 정보:**

- 테스터 모집
- 세일 정보 (실, 도안 등)
- 팝업 / 행사

### 탭3: 데이터 (Materials & Patterns)

**구조:**

- 실 정보
- 도안 정보
- 후기 및 평가
- 게이지 정보
- 실 ↔ 도안 연결 (다대다 구조)

**기능:**

- 특정 실로 뜬 작품 조회
- 특정 도안에 사용된 실 조회
- 사용자 후기 기반 정보 축적

### 탭4: 커뮤니티 (추후)

- 함뜨 기능 (같이 뜨기)
- 일정 기반 참여
- 같은 도안을 뜨는 사용자 연결

## 5. 핵심 기능 (MVP)

### 5.1 지도 기반 탐색

- 첫 화면에서 바로 지도 노출
- 장소 마커 표시
- 마커 클릭 시 정보 카드 표시

### 5.2 지역 필터

- 전체 / 서울 / 경기 / 인천 / 강원 / 충청 / 경상 / 전라 / 제주
- 선택 시 지도 중심 이동 및 줌 변경

### 5.3 카테고리 필터

- 뜨개샵
- 공방
- 뜨개카페
- 손염색실
- 공예용품점

### 5.4 검색 기능

- 상호명
- 주소
- 지역 기반 검색

### 5.5 장소 정보 카드

- 이름
- 카테고리
- 주소
- 운영시간
- 상태
- 한줄 설명
- 태그

### 5.6 상태 표시

- 운영중
- 이전
- 폐업/종료
- 확인 필요

### 5.7 지도-리스트 동기화

- 마커 클릭 → 카드 선택
- 카드 클릭 → 지도 이동

## 6. 사용자 역할 및 권한

### 6.1 비회원 (Guest)

- 조회만 가능
- 지도, 장소, 이벤트 확인 가능

### 6.2 일반 사용자 (User)

- 이벤트 등록 가능
- 데이터 등록 가능
- 커뮤니티 참여 (추후)

### 6.3 관리자 (Admin)

- 제보 승인 / 반려
- 데이터 직접 수정
- 상태 관리
- 사용자 관리

---

## 7. 인증 및 인가 구조

### 인증

- 소셜 로그인 기반 (추후)
- 초기 MVP는 선택적 로그인

### 인가

모든 사용자 생성 데이터는 다음 흐름을 따른다.

```
제보 → pending → 관리자 승인 → approved → 서비스 노출
```

**상태:**

- `pending`
- `approved`
- `rejected`
- `hidden`
