# 타래 기능 명세서 (Final)

---

## 1. 서비스 구조 개요

타래는 다음 4개의 핵심 탭으로 구성된다.

| 탭 | 이름 | 핵심 질문 |
|----|------|-----------|
| Tab 1 | **Places** (지도) | 어디서 |
| Tab 2 | **Events** (캘린더) | 언제 |
| Tab 3 | **Data** (실/도안) | 무엇을 |
| Tab 4 | **Community** (함뜨) | 누구와 |

---

## 2. Tab 1: Places (지도)

### 2.1 목적

사용자가 뜨개 관련 장소를 지도 기반으로 탐색할 수 있도록 한다.

### 2.2 주요 기능

#### 2.2.1 지도 초기 화면

- 첫 진입 시 지도 바로 노출
- 전국 기준 중심 좌표 설정
- 기본 마커 데이터 로딩

#### 2.2.2 마커 표시

- 장소 데이터를 기반으로 지도에 마커 렌더링
- 마커 클릭 시 해당 장소 선택

#### 2.2.3 지역 필터

**지역 목록:**

전체 / 서울 / 경기 / 인천 / 강원 / 충청 / 경상 / 전라 / 제주

**동작:**

- 지도 중심 이동 (pan)
- 줌 레벨 변경
- 해당 지역 데이터만 표시

#### 2.2.4 카테고리 필터

**카테고리:**

- 뜨개샵
- 공방
- 뜨개카페
- 손염색실
- 공예용품점

- 다중 선택 가능
- 지도와 리스트 동기화

#### 2.2.5 검색

- 상호명 / 주소 / 지역 기반 검색
- 입력 시 debounce 처리
- 결과 기반 지도 이동

#### 2.2.6 장소 카드 (Bottom Sheet)

**구성:**

| 항목 | 설명 |
|------|------|
| 이름 | 장소명 |
| 카테고리 | 분류 |
| 주소 | 위치 정보 |
| 운영시간 | 영업 시간 |
| 상태 | 운영 상태 |
| 한줄 설명 | 간단 소개 |
| 태그 | 키워드 |

**동작:**

- 마커 클릭 시 표시
- 카드 클릭 시 상세 확장

#### 2.2.7 상태 표시

| 상태 | 설명 |
|------|------|
| 운영중 | 정상 운영 |
| 이전 | 위치 변경 |
| 폐업 | 영업 종료 |
| 확인 필요 | 미확인 상태 |

- 색상 및 배지로 구분

#### 2.2.8 지도-리스트 동기화

- 마커 클릭 → 리스트 선택
- 리스트 클릭 → 지도 이동
- 선택된 항목 강조

#### 2.2.9 장소 제보

**입력:**

- 이름
- 주소
- 카테고리
- 설명

**상태:**

- `pending` → 승인 후 노출

## 3. Tab 2: Events (캘린더)

### 3.1 목적

뜨개 관련 일정 및 모집 정보를 캘린더 기반으로 탐색 및 참여할 수 있도록 한다.

### 3.2 구조

이벤트는 3가지 유형으로 구성된다.

| 유형 | 성격 |
|------|------|
| 테스터 모집 | 참여형 기능 |
| 세일 정보 | 큐레이션 |
| 행사 / 팝업 | 정보형 |

### 3.3 공통 기능

#### 3.3.1 캘린더 뷰

- 월 단위 캘린더
- 날짜별 이벤트 표시
- 선택 날짜 하이라이트

#### 3.3.2 이벤트 필터

- 테스터 모집
- 세일 정보
- 행사 / 팝업
- 다중 선택 가능

#### 3.3.3 이벤트 리스트

- 선택 날짜 기준 리스트 표시
- 구성: 제목, 유형, 기간, 상태

#### 3.3.4 이벤트 상세

- 제목
- 설명
- 일정
- 링크
- 위치 (optional)

### 3.4 테스터 모집 (핵심 기능)

#### 3.4.1 목적

모집자가 직접 모집을 등록하고, 사용자가 서비스 내에서 신청할 수 있도록 한다.

#### 3.4.2 주요 흐름

```
모집 등록 → 모집 노출 → 사용자 신청 → 신청자 관리 → 모집 마감
```

#### 3.4.3 모집 등록

**입력:**

| 항목 | 설명 |
|------|------|
| 제목 | 모집 제목 |
| 도안명 | 테스트 대상 도안 |
| 카테고리 | 분류 |
| 모집 설명 | 상세 내용 |
| 모집 인원 | 정원 |
| 신청 시작일 / 마감일 | 신청 기간 |
| 테스트 기간 | 테스트 수행 기간 |
| 조건 | 참여 조건 |
| 제출 요구사항 | 결과물 요구 |
| 연락 방식 | 소통 채널 |
| 링크 | 외부 링크 |

**상태:**

- `pending` → `approved` → `open`

#### 3.4.4 모집 상태

| 상태 | 설명 |
|------|------|
| `draft` | 작성 중 |
| `pending` | 승인 대기 |
| `open` | 모집 중 |
| `closed` | 모집 마감 |
| `in_progress` | 테스트 진행 중 |
| `completed` | 완료 |
| `hidden` | 숨김 |

#### 3.4.5 신청 기능

**조건:**

- 로그인 필요
- `open` 상태
- 마감일 이전
- 중복 신청 불가

**입력:**

| 항목 | 필수 |
|------|------|
| 닉네임 | O |
| 연락처 | O |
| 경험 | O |
| 지원 이유 | O |
| 포트폴리오 | X (optional) |

#### 3.4.6 신청 상태

| 상태 | 설명 |
|------|------|
| `submitted` | 신청 완료 |
| `reviewing` | 검토 중 |
| `accepted` | 수락 |
| `rejected` | 거절 |
| `canceled` | 취소 |

#### 3.4.7 신청자 관리

모집자가 수행:

- 신청자 목록 조회
- 상태 변경 (`accepted` / `rejected`)

#### 3.4.8 모집 마감

- `open` → `closed`
- 필요 시 `in_progress` 전환

#### 3.4.9 캘린더 연동

- 모집 일정은 캘린더에 표시
- 신청 기간 기준 노출

## 4. Tab 3: Data (실 / 도안)

### 4.1 목적

뜨개 데이터를 구조화하여 검색 및 탐색 가능하게 한다.

### 4.2 주요 기능

#### 4.2.1 실 목록

- 이름
- 브랜드
- 특징

#### 4.2.2 도안 목록

- 이름
- 난이도
- 카테고리

#### 4.2.3 실 상세

- 실 정보
- 후기
- 사용된 도안
- 작품 리스트

#### 4.2.4 도안 상세

- 도안 정보
- 사용된 실
- 후기
- 결과물

#### 4.2.5 후기 시스템

**입력:**

- 내용
- 게이지
- 평점

#### 4.2.6 관계 구조

```
실 ↔ 도안  (다대다)
실 ↔ 후기  (일대다)
도안 ↔ 후기 (일대다)
```

#### 4.2.7 검색

- 이름 검색
- 태그 기반 검색

#### 4.2.8 데이터 제보

- 실 / 도안 등록
- `pending` → 승인

## 5. Tab 4: Community (함뜨)

### 5.1 목적

사용자 간 협업 및 연결 경험 제공

### 5.2 주요 기능

#### 5.2.1 함뜨 생성

**입력:**

- 도안
- 시작일
- 설명

#### 5.2.2 함뜨 참여

- 참여 버튼
- 참여자 목록

#### 5.2.3 일정 연동

- 캘린더와 연결

#### 5.2.4 커뮤니케이션 (추후)

- 댓글
- 진행 공유

---

## 6. 공통 시스템

### 6.1 제보 시스템

- 모든 데이터는 `pending` 상태로 등록
- Admin 승인 후 노출

### 6.2 상태 관리

| 상태 | 설명 |
|------|------|
| `pending` | 승인 대기 |
| `approved` | 승인 완료 |
| `rejected` | 반려 |
| `hidden` | 숨김 |

### 6.3 사용자 권한

| 역할 | 권한 |
|------|------|
| **Guest** | 조회 |
| **User** | 제보, 신청 |
| **Admin** | 승인, 수정 |

## 7. MVP 범위

### 포함

- 지도 (Places)
- 필터 / 검색
- 장소 데이터
- 캘린더 기본 구조
- 테스터 모집 (등록 + 신청)

### 제외

- 커뮤니티 상세 기능
- 고급 알림 시스템
- 자동 추천 시스템

---

## 최종 정리

타래는 다음 4가지 축으로 구성된 플랫폼이다.

- **Places** — 장소 탐색
- **Events** — 일정 및 모집
- **Data** — 실과 도안 정보
- **Community** — 사용자 연결

특히 **테스터 모집 기능**을 통해 단순 정보 제공을 넘어 실제 사용자 행동이 발생하는 서비스로 확장한다.
